# Watch platform — v0.3

## Summary

v0.2 turned the live estate into a **gated lifecycle** — deterministic create/teardown, a
SAST→DAST→smoke chain, HTTPS hardening — but left one thing stubbed: the app emitted telemetry to
a **local Alloy sidecar that forwarded to a debug sink**. The backend didn't exist yet.

**v0.3 completes the observability plane end-to-end.** Telemetry now flows the whole way —
**app → sidecar → per-env gateway → a real backend** — with the backend chosen by IaC per
environment exactly as ADR-016 promised: **staging → an in-AWS Tempo + Grafana slice**,
**prod → Grafana Cloud** (managed vendor). The gateway does **incident-tuned tail-sampling** (keep
every error, slow trace, and state transition; sample the boring reads), and signal routing is
**backend-aware** (a traces-only backend doesn't get metrics/logs it would reject). The whole
thing runs **warm-minimal** rather than destroy/recreate, because it's now part of the release
path. And the repos are **MIT-0 licensed** for frictionless reuse.

The through-line is **realizing the backend-agnostic promise**: v0.2 proved the app names no
backend; v0.3 plugs two very different backends into the *same* app config — an in-VPC Tempo over
gRPC and a vendor over authenticated OTLP/HTTP — with **zero app change**, only collector config.

Delivered: **#19, #29, #23, #38**, plus MIT-0 licensing and the observability architecture doc
(13 platform + 3 watch commits since `v0.2.0`).

## At a glance

| Area | v0.2 | v0.3 |
|---|---|---|
| Telemetry destination | sidecar → **debug sink** (no backend) | sidecar → gateway → **real backend**, per env |
| Staging backend | — | **Tempo + Grafana** slice, co-located in the staging VPC |
| Prod backend | — | **Grafana Cloud** (managed vendor), OTLP/HTTP + auth |
| Gateway | not deployed | per-env **Alloy collector** (ECS, Cloud Map `gateway.<env>.svc`) |
| Sampling | none | **incident-tuned tail-sampling** (keep errors/slow/writes, sample reads) |
| Signals | traces only, implicitly | traces everywhere; **metrics+logs to Grafana Cloud**, dropped for Tempo |
| Vendor creds | — | from `.env` → **TF-managed SSM SecureString**, no imperative CLI |
| Licensing | none | **MIT-0** across watch / platform / watchtower |
| Teardown hygiene | Lambda log groups orphan every cycle | **declared in TF** (#38) — destroy cleans them |
| Cost posture | staging ephemeral (destroy between releases) | staging + slice **warm-minimal** (scale to 0 idle) |

## Capabilities: how they were built and tested

### 1. The per-env telemetry gateway (#19)

**Built.** `modules/gateway` is a shared Alloy collector as an ECS Fargate service with its own
cluster and a **Cloud Map** private DNS namespace, registering as `gateway.watch-<env>.svc:4317`
so the app sidecars resolve it. Its SG admits only the app SG on the OTLP ports; it follows the
env cost profile (private-subnet+NAT for prod, public for lean staging). The config comes from a
**shared renderer** (`modules/alloy`, `sidecar`/`gateway` roles) with three exporter modes —
vendor (OTLP/HTTP+TLS), in-VPC gRPC, or a debug sink — so the pipeline is valid before any
backend exists.

**Tested.** Staging gateway task came up 1/1 with the River config loading clean (receiver →
batch → exporter all evaluated, `:4317`/`:4318` listening); Cloud Map resolved
`gateway.watch-staging.svc` to the task IP.

### 2. The staging Tempo + Grafana slice (#29)

**Built.** Rewritten from the never-applied `~/watchtower` Terragrunt draft to align with platform
(env→file config, **no baked images / S3 / EFS / VPC peering**). `modules/tempo` runs Grafana
Tempo — its distributor takes OTLP directly, local trace store, query API `:3200` — and owns the
shared obs cluster + `watch-obs.svc` namespace. `modules/grafana` runs Grafana behind a small
public ALB with the Tempo datasource provisioned and the admin password generated into SSM.
**Co-located in the staging VPC** (both are warm-standby now) → no peering (a deliberate,
ADR-documented deviation from the separate-account vision, seam preserved).

**Tested.** End-to-end on staging: real `watch-backend` traces (HTTP + DB `SELECT` spans) flowed
app → sidecar → gateway → Tempo and rendered in **Grafana Explore via TraceQL** — verified in a
browser (Playwright), not just curl. Grafana→Tempo connectivity proven independently
(`/api/echo` → `echo`).

### 3. Prod → Grafana Cloud (managed vendor, #19)

**Built.** Per ADR-016 §2 / ADR-018, product prod goes to a **managed vendor, never the Watchtower
slice**. `prod/gateway` (ha, private subnets, NAT egress) exports to Grafana Cloud, consuming
exactly what its OTLP connection emits — `OTEL_EXPORTER_OTLP_ENDPOINT` +
`OTEL_EXPORTER_OTLP_HEADERS` — from `~/platform/.env` via `get_env`. The **token becomes a
TF-managed SSM SecureString** (a sensitive var → `apply`, mirroring the `random_password` secrets
pattern — *not* an imperative `aws ssm put-parameter`). The vendor's `Authorization` header is
URL-encoded (`Basic%20…`), so terragrunt strips `Authorization=` and decodes `%20`→space.

**Tested.** A direct OTLP `POST /v1/traces` with the parsed header returned **HTTP 200**; the same
without auth returned **401** — proving the creds/header parsing independent of the gateway. The
prod gateway task then loaded `otelcol.exporter.otlphttp.dest` (→ Grafana Cloud) cleanly. (Real
prod app traces await a prod promotion — the running tasks only forward after CodeDeploy rolls the
gateway-forwarding task def.)

### 4. Incident-tuned tail-sampling (#23)

**Built.** The gateway policy (ADR-016 §3) keeps a trace on **any** match: `errors` (ERROR spans),
`slow` (over the latency floor), `writes` (`http.method` ∈ POST/PUT/PATCH/DELETE — **every
ack/escalate/resolve/intake**, the state transitions you replay in a postmortem), else a
probabilistic slice of the boring reads. Enabled on **both** gateways (staging rehearses prod);
traces route through the sampler, metrics/logs bypass it.

**Tested.** Enabling it surfaced **two real bugs in dormant code** (never runtime-tested before):
compact one-line River `policy {}` blocks are invalid (`expected TERMINATOR`) → expanded to
multi-line and **validated against the real `grafana/alloy` binary** before deploy; and Tempo is
**traces-only** and was rejecting the app's metrics+logs (`Unimplemented`) → added
`dest_traces_only` to drop them at the receiver for Tempo while Grafana Cloud still gets all three.
The policy was then proven on staging: in a 34s window, **6/6 POST writes kept, 4/50 health reads
kept (~8%)**.

### 5. MIT-0 licensing

**Built.** `LICENSE` (MIT No Attribution) + a README license note across **watch, platform, and
watchtower**. MIT-0 is OSI-approved, keeps the standard liability disclaimer, and drops the
attribution requirement — the least-friction permissive license for downstream/government reuse.

### 6. Teardown hygiene — Lambda log groups (#38)

**Built.** Lambda auto-creates `/aws/lambda/<fn>` outside Terraform, so `destroy` never removed
them and every teardown orphaned `watch-*-deploy-hook` + intake authorizer/consumer log groups for
the sweep. Now each Lambda declares its `aws_cloudwatch_log_group` (escalation already did; added
intake authorizer + consumer and the pipeline deploy-hook `for_each`), so the managed group is
reused and destroy removes it. Takes effect on the next clean create.

**Tested.** Both stacks validate; GOTCHAS updated with the one-final-sweep caveat.

## Operational reference

```
# verify vendor creds without any traffic (200 = good, 401 = auth wrong)
set -a; . ./.env; set +a
AUTH="${OTEL_EXPORTER_OTLP_HEADERS#Authorization=}"; AUTH="${AUTH//\%20/ }"
curl -s -o /dev/null -w '%{http_code}\n' -X POST "$OTEL_EXPORTER_OTLP_ENDPOINT/v1/traces" \
  -H "Authorization: $AUTH" -H 'content-type: application/json' --data '{"resourceSpans":[]}'

# query staging traces:  Grafana → Explore → Tempo → TraceQL { resource.service.name = "watch-backend" }
# validate an Alloy config before deploy (compact policy blocks are invalid River):
docker run --rm -v "$PWD:/c" grafana/alloy validate --stability.level=experimental /c/config.alloy
# scale a warm gateway to 0 when idle (ignore_changes → no Terraform fight):
aws ecs update-service --cluster watch-<env>-gateway --service watch-<env>-gateway --desired-count 0
```

Full plane + runbook: **[`../architecture/observability.md`](../architecture/observability.md)**.

## ADR trail

- **ADR-016** amended — plane deployed & verified; staging → Tempo slice, prod → Grafana Cloud;
  §3 tail-sampling policy made concrete (incident-tuned).
- **ADR-018** amended — the Watchtower slice is realized as platform modules **co-located in the
  staging VPC** (single-account DEBT applied to the platform plane; account seam preserved).
- **ADR-019** amended — staging + the slice are **warm-standby** (scale-to-minimal), not
  destroy/recreate, since they now serve continuous CI/CD.

## Gotchas

The traps found building v0.3 — Alloy River needs one attribute per line, Tempo is traces-only,
Grafana Cloud's OTLP header is `%20`-encoded, SG descriptions must be ASCII, empty
`health_check_custom_config {}` forces a Cloud Map replace — are in
[`../../GOTCHAS.md`](../../GOTCHAS.md) and the observability doc.
